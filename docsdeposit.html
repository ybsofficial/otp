X-API-KEY: your_api_key_here

Flow Deposit QRIS

1

User request create_deposit melalui API.

2

Sistem JASA OTP melakukan generate Dynamic QRIS.

3

User melakukan scan dan pembayaran sukses.

4

Jika status SUCCESS, saldo user otomatis bertambah secara instan.

Create Deposit

GET /api.php

Endpoint ini digunakan untuk membuat tagihan deposit baru dengan output string QRIS atau URL QR.

Parameters:

amount Required | Integer

Contoh Request:

?action=create_deposit&amount=10000

JSON Response:

{
  "success": true,
  "data": {
    "deposit_id": "DEP123456",
    "amount": 10000,
    "qr_url": "https://qris.link/xxxxx",
    "expired_at": "2026-01-25 12:00"
  }
}

Check Deposit Status

GET /api.php

Lakukan polling ke endpoint ini untuk memantau apakah pembayaran sudah masuk.

Sistem Perilaku: Jika status success, saldo user otomatis bertambah dan notifikasi dikirim ke Telegram Admin. 

?action=cek_deposit&id=DEP123456

IF PENDING:

{"success": true, "data": {"status": "pending"}}

IF SUCCESS:

{"success": true, "data": {"status": "success"}}

Deposit Status List

Status Description
pending Menunggu pembayaran dari user
success Pembayaran diterima, saldo telah ditambahkan
canceled Deposit dibatalkan otomatis atau manual

Notes & Security

  • Setiap deposit hanya diproses satu kali (Anti-Double Processing).
  • Sistem melakukan validasi X-API-KEY dan Session secara ketat.
  • Semua aktivitas transaksi tercatat di sistem log database dan dipantau via Telegram.
